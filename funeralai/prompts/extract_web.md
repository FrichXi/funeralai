# 信息提取（v3 — 含产品体验实查）

从材料中提取结构化信息。只提取，不判断。

材料包含两部分：**页面内容**（网站正文提取）和**产品体验实查报告**（自动生成的网站检查结果）。分别提取两类证据，并交叉比对声称与现实。

用 JSON 格式输出，不要输出任何 JSON 之外的内容：

```json
{
  "material_type": "产品评测 / 行业观察 / 广告软文 / 方法论 / 专访 / 整活 / 产品网站 / 其他",
  "products": ["涉及的产品名"],
  "author_attitude": "正面 / 负面 / 中性 / 混合 / 不明确",
  "attitude_signals": ["支撑态度判断的关键表述，如标题用词、反复出现的情绪词、收尾定性"],
  "facts": ["来自页面内容的事实，如用户数、功能描述、定价、时间线"],
  "opinions": ["页面中的主观判断和评价"],
  "gaps": ["材料未覆盖但对投资判断重要的信息"],
  "key_quotes": ["最关键的原文引用，不超过5条"],
  "product_evidence": ["来自体验实查报告的硬证据，如响应时间、JS 错误数、交互元素数量、链接健康度、资源加载情况"],
  "claim_vs_reality": [
    {"claim": "页面中的声称", "reality": "体验检查的实际发现", "gap": "差距描述"}
  ]
}
```
