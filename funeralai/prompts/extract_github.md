# 信息提取（v2 — 含代码实查）

从材料中提取结构化信息。只提取，不判断。

材料包含两部分：**项目描述**（README 或文章）和**技术实查报告**（自动生成的代码检查结果）。分别提取两类证据，并交叉比对声称与现实。

用 JSON 格式输出，不要输出任何 JSON 之外的内容：

```json
{
  "material_type": "产品评测 / 行业观察 / 广告软文 / 方法论 / 专访 / 整活 / GitHub仓库 / 其他",
  "products": ["涉及的产品名"],
  "author_attitude": "正面 / 负面 / 中性 / 混合 / 不明确",
  "attitude_signals": ["支撑态度判断的关键表述，如标题用词、反复出现的情绪词、收尾定性"],
  "facts": ["来自文本描述的事实，如用户数、功能描述、时间线"],
  "opinions": ["作者的主观判断和评价"],
  "gaps": ["材料未覆盖但对投资判断重要的信息"],
  "key_quotes": ["最关键的原文引用，不超过5条"],
  "code_evidence": ["来自技术实查报告的硬证据，如实际代码行数、测试覆盖、贡献者分布、代码采样观察"],
  "claim_vs_reality": [
    {"claim": "文本中的声称", "reality": "代码检查的实际发现", "gap": "差距描述"}
  ]
}
```
