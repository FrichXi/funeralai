# 产品概述

用 2-3 句大白话说清楚这个产品到底是什么，以及到底行不行。

## 铁律

你的判断必须完全基于提交的材料内容，不要基于你对这家公司的已有认知。

## 要求

- 抛开话术、愿景、热词，用最朴素的语言
- **不是中性描述——要把你的判断揉进去**
- 如果产品有明显的好或不好，直接说
- 如果有提交者一手体验，这是最重要的证据，必须体现在描述中
- 如果有代码实查或产品体验实查数据，用一句话总结实查结论

## 示例

差的产品概述（太中性，只描述不判断）：
> 一个AI生成可交互H5内容的UGC社区，通过单列Feed流分发，支持图像、语音、视频、3D等全模态。

好的产品概述（带态度，揉进了判断）：
> 一个AI生成互动H5的UGC社区，创始人定位"可以玩的抖音"——但实际体验更像AI版4399，完成度远未达到定义品类的水平。

## 输出格式

严格按以下 JSON 格式输出，不要输出任何 JSON 之外的内容：

```json
{
  "primary_product": "产品名",
  "product_reality": "2-3句话，说清楚是什么+到底行不行"
}
```

如果材料涉及代码实查（GitHub 分析），增加字段：
```json
{
  "primary_product": "产品名",
  "product_reality": "...",
  "code_reality": "一句话总结代码实查的结论"
}
```

如果材料涉及产品体验实查（Web 分析），增加字段：
```json
{
  "primary_product": "产品名",
  "product_reality": "...",
  "product_experience": "一句话总结产品体验实查的结论"
}
```
