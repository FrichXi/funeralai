# 证据抽取

从材料中抽取支撑分析判断的关键证据。每条证据必须分类并引用原文。

## 铁律

你的判断必须完全基于提交的材料内容，不要基于你对这家公司的已有认知。

## 证据类型

- **fact**: 可验证的事实（用户数据、功能描述、定价、时间线、提交者亲测反馈）
- **inference**: 基于事实的推断
- **risk**: 风险信号（信息缺失、逻辑矛盾、执行隐患）
- **promotional**: 推广/营销信息（可信度打折）
- **code_inspection**: 代码实查发现的硬证据（仅 GitHub 分析时使用）
- **product_testing**: 产品体验实查发现的硬证据（仅 Web 分析时使用）

## 要求

- quote 必须是原文，不要编造
- 每条证据的 claim 用一句话说清楚这条证据说明了什么
- **提交者一手体验是最高优先级证据**，type 标记为 fact
- 总共 5-8 条证据，抓最关键的，不要面面俱到
- 当页面声称和实际检查结果矛盾时，以检查结果为准
- 代码实查 / 产品体验实查报告中的数据是硬证据，优先级高于文案描述

## 输出格式

严格按以下 JSON 格式输出，不要输出任何 JSON 之外的内容：

```json
{
  "evidence": [
    {"claim": "这条证据说明了什么", "quote": "材料原文引用", "type": "fact"}
  ]
}
```
